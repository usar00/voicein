## なにこれ

Linux（Wayland / GNOME）向けに、**OpenAIのAPIキーを登録するだけで使える音声入力ツール**を作りました。ほぼキーボードを使わずに、話した内容をそのまま、いま開いているアプリへ入力できます。

- 指定したキーを押している間だけ録音し、離すと変換します
- OpenAIの `gpt-4o-transcribe` で変換します（句読点も自然に付きます。日本語も高い精度で、`language` 設定で他の言語にも対応できます）
- 変換したテキストを、いま開いているアプリ（Claude / ブラウザ / ターミナルなど）のカーソル位置へ自動で挿入します
- 変換履歴を一覧で見られて、ボタンひとつでコピーできるGUIも付けています

ローカルにモデルを用意するような手間はいりません。APIキーをひとつ登録すれば動きます。

リポジトリはこちらです。MITライセンスですので、どなたでも自由に使っていただけます。

https://github.com/usar00/voicein

ちなみにこの記事も、ほとんどこのツールで話しながら書いています。

## なぜ作ったのか

音声入力は、慣れるととても快適です。手を止めずに文章を書けるので、一度使うと手放せなくなります。

音声入力ツールはすでにいくつもあります。たとえば Wispr Flow は月額 $15 ほど、superwhisper は月額 $8.49 ほど（買い切り $249.99）といった価格です（記事執筆時点）。多くはサブスクリプションが前提で、使う頻度にかかわらず毎月一定額がかかります。また、ローカルにモデル（Whisperなど）を立てる方法は無料に近いものの、環境構築やGPUの用意など、最初のひと手間がかかります。

そこで私は、**「APIキーを登録するだけで、使った分だけ支払う」**という形がちょうど良いと感じました。普段の作業環境がLinuxなので、そこで気持ちよく音声入力ができると、日々の体験がかなり変わります。

幸い、いまは音声認識のAPIがとても手の届きやすい価格になっています。`gpt-4o-transcribe` で音声1分あたり $0.006（約0.9円）、軽量版の `gpt-4o-mini-transcribe` なら $0.003（約0.45円）です。新規アカウントには $5 ほどの無料枠もあり、これだけで十数時間分を試せます。

精度については、この記事を最初から最後まで音声入力で書ける程度には実用的でした。完璧ではなく、固有名詞などは時々間違えますが、そこは認識のヒント（`prompt`）に単語を登録して補えます。マイクの近さや静かさでも体感は変わるので、このあたりは使いながら詰めていく感じです。

個人が、自分の使いたい道具を、自分の手で組み立てられる。そういうことが現実的にできる時代になったのだと、作ってみて改めて感じました。サブスクリプションの代わりになる選択肢のひとつとして、気軽に使ってもらえたら嬉しいです。

## 仕組み

全体の流れはシンプルです。

```
[キーを押している間] parecord で録音(16kHz mono WAV)
     │  キーを離す
     ▼
OpenAI gpt-4o-transcribe で変換
     │
     ▼
wl-copy でクリップボードへ → 仮想キーボード(evdev/uinput)で
Ctrl+V を送出 → カーソル位置に貼り付け
     │
     ▼
履歴(JSONL)に保存 → GUIで一覧・コピー
```

Wayland（GNOME）はアプリ間の合成入力を塞いでいるので、`xdotool` のような定番ツールは効きません。代わりにカーネルの `uinput` で仮想キーボードを1つ作り、「物理キーボードがもう1台挿さった」状態にして入力を送っています。日本語は物理キーで打てないので、テキストはクリップボード経由で `Ctrl+V` 貼り付けです。

## いちばん悩んだのは「どのキーで録音するか」だった

技術的に一番てこずったのは、意外にも**プッシュトゥトークのキー選び**でした。

押している間だけ録音したいので、キーの押下と解放を監視します。ところがWaylandでは、こちらがキーを監視しても、そのキー入力は**裏で開いているアプリにもそのまま流れていきます**。つまり、もし録音キーをスペースや適当な文字キーにすると、しゃべるたびに下のアプリにもその文字が打ち込まれてしまう。これは困る。

「監視したキーを横取りして、下に流さない」ことも技術的には可能です。ただしそれをやると、そのキーボード**全体**が録音中は効かなくなる（1キーだけ横取り、ということができない）。これも論外でした。

そこでたどり着いた答えが、**「単体で押しても何も起きないキー」を録音キーにする**こと。右Ctrlや右Shiftは、単独で押しても普通のアプリでは何も起こりません。だから録音中に裏へ流れても無害です。さらにこれを利用して、

- **右Ctrl** … 通常のアプリ用（貼り付けは `Ctrl+V`）
- **右Shift** … ターミナル用（貼り付けは `Ctrl+Shift+V`）

と、キーごとに貼り付け方法まで変えました。ターミナルだけ貼り付けキーが違う問題も、これで同時に解決です。USBのフットペダル（独自の未使用キーを送る）も同じ理屈でそのまま使えます。

```ini
[bindings]
KEY_RIGHTCTRL = ctrl+v          ; 通常のアプリ(Claude / ブラウザ / 入力欄)
KEY_RIGHTSHIFT = ctrl+shift+v   ; ターミナル
```

## 使いながら足していった工夫

一通り動いてから実際に使ってみると、細かい「気になる」が出てきて、そのたびに直していきました。

**「押した瞬間にピコッと鳴る」のがうるさかった。** 右Ctrlに軽く触れただけで録音開始音が鳴ってしまう。そこで **0.15秒の「溜め」** を入れ、しっかり押し続けたときだけ録音を始めるようにしました。ちょっと触れただけの誤爆は、音も録音も出ません。たった0.15秒ですが、操作感がだいぶ落ち着きました。

**しゃべった内容を後で使いたい。** 音声入力って、一度しゃべった文章をもう一回使いたくなることが結構あります。そこで変換結果をすべて履歴に残し、**一覧で見られて、各行のボタンや行クリックでコピーできるGUI** を付けました。新しく入力するとリストも自動で増えます。「さっき入れたやつ、もう一回」がワンクリックでできます。

このあたりは最初から設計していたというより、**使って・気になって・直す**を繰り返して固まっていった部分です。

## 使い方

```bash
git clone https://github.com/usar00/voicein.git
cd voicein
bash install.sh        # 依存の導入・権限設定・venv作成
# 一度ログアウト/ログイン(inputグループ反映のため)
# config.ini に OpenAI APIキーを設定
./run.sh               # 起動
```

あとは右Ctrlを押しながら話し、離すだけです。カーソル位置に文字が入ります。

```bash
./history.sh           # 変換履歴ビューア(一覧・コピー)
```

ログイン時の自動起動（systemd user service）も同梱しています。

## 動作環境

- Linux（**Wayland / GNOME / Ubuntu** で開発・確認しています。X11でも動作する設計です）
- 音声: PulseAudio または PipeWire（`parecord`）
- OpenAI APIキー（従量課金。`gpt-4o-transcribe` で約 $0.006/分）

`evdev`/`uinput` などLinux固有の仕組みを使っているため、現状はLinux向けです。他ディストロでも、パッケージ名を読み替えればおおむね動作します。

## 補足：OpenAIの課金について

ひとつだけ事前の注意点です。OpenAIは前払い式のため、APIキーを作成しただけでは `insufficient_quota` となり動きません。先にクレジットの購入が必要です。

また、決済時に **JCBカードは通らないことが多い**です。私自身もJCBで失敗し、Visa/Mastercard（デビットでも可）に変更したところ通りました。もし同じところで止まったら、カードを変えてみてください。
